## Key Changes from Previous Run

### Driver Update

**Old pattern** (HTTP query param):
```
http://127.0.0.1:PORT/index.html?protocol=mtt_reagent_prep
```

**New pattern** (per-protocol file:// URL):
```
file:///path/to/dist/mtt_reagent_prep.html
```

Per-protocol HTML files are generated by the foundation build and contain:
- Inlined protocol name in a `<script type="application/json">` tag
- Import of shared `dist/runtime.bundle.js` (IIFE format)
- Bootstrap script that calls `SceneRuntime.loadAndMountByProtocolName(root, name)`

### Mount Mechanism

Each per-protocol HTML file is self-contained:
1. Loads `window.SceneRuntime` from `runtime.bundle.js`
2. Calls `loadAndMountByProtocolName(root, protocol_name)`
3. Sets `window.__RUNTIME_PROTOCOL_CONFIG` with full world state
4. Renders scene and interaction prompt

All 11 protocols mount cleanly on first load. No app-layer protocol picker UI required; each protocol is its own entry point.
