Panel 4. Diagnostics evidence integrity: INVALID vs VALID

INVALID evidence patterns
What does NOT count as evidence.
  • precheck.mjs with a +243-line local edit that loosens assertions to make the run pass -- the diagnostic was rewritten, not satisfied.
  • render_and_dump.mjs with a regex that strips DOM nodes before the dump runs -- the dump no longer reflects the rendered page.
  • Bench-side hand-edited audit JSON to remove hard-fail entries before scoring.
These are tampered diagnostics: they change the measurement instrument to fit the desired outcome.
VALID evidence patterns
What does count as evidence.
  • Canonical precheck.mjs at the committed revision, run against the working tree without local edits.
  • Bridge-integrity guardrail asserts that placement-count cannot decrease between consecutive audit runs -- regressions surface immediately.
  • render_and_dump.mjs emitting the page as rendered, with the audit operating on that unmodified dump.
These keep the instrument fixed so score deltas reflect the scene, not the harness.
