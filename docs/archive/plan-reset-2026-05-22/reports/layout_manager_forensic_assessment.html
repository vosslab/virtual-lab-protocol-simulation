Layout Manager Forensic Assessment

Subject: Experiments vs Runtime vs Salvage — what currently ships, what is dead code, what is unpromoted value.

Date: 2026-05-22 · Repo: virtual-lab-protocol-simulation · Author: forensic assembly pass (FH)

Inputs: FA provenance, FB experiments-value audit, FC salvage-contamination audit, FD SVG completeness, FF side-by-side, FG diagnostic measurements, plus the layout_manager_forensic_gallery (45 PNGs) and runtime_svg_completeness gallery (per-protocol crops).

1. Executive summary

Headline finding: both layout paths fail their own quality bars. The shipping procedural runtime exhibits 17.3% hard fails on SVG completeness (48/278 placements) and only 12 of 26 protocols complete an end-to-end walker pass. The CSS-native experiments path scores 0/10 PASS on its own precheck and 0/110 on its own stress corpus. There is no clean winner today; there is the path we ship and the path we did not.

The user's standing suspicion — that "broken salvage code has come back into src/" — is REFUTED. src/scene_runtime/layout/layout_engine.ts was mined verbatim from src/layout_engine.ts (commit ff8c664, 2026-05-17), which itself dates to 41a236d (2026-05-05). The salvage/ tree contains only Python utilities and agent-transcript text; zero TypeScript source files. experiments/css_native_layout/ is bidirectionally isolated from src/: it explicitly imports nothing from src/, and git ls-files src/ shows zero files referencing any experiments/ path. The one apparent exception — src/scene_runtime/layout/css_native_adapter.ts — is a fresh spike (2026-05-21) authored inside src/, feature-flagged off, and its own header reads "Must be removed or replaced before NEW1 promotion."

The runtime side ships a zone-based pixel-math layout engine plus a working dispatch and chrome stack. It mounts 26/26 protocols cleanly and renders 230 real SVG assets out of 278 placements (82.7%). But 48 placements fall into generic_outline (42, mostly instrument off-states such as power_supply_off, microwave_closed, heat_block_closed, lightbox_off, and the recurring electrode_module / gel_opening_tool) or empty_bubble (6, all on well_plate_96). The walker completes only 12 of 26 protocols end-to-end; failure modes are split across "missing DOM affordance," "visible-but-timed wait without test-clock affordance," and "next-button gating after step_validator." None of these failures are mount failures.

The experiments side never got close to shipping. Its own precheck verdict (visual_audit.json) marks all ten gold templates FAIL, dominated by artwork_integrity findings that the precheck design also classifies as hard-fail. The stress corpus is worse (scenes_pass=0/110, checks_failed=216). The verdict-vs-score disagreement is by design: even well_plate_96_zoom at score 92 is FAIL. Where the experiments side did add durable value is in its diagnostic vocabulary (checkArtworkIntegrity, checkRegionOverflow, checkSvgSvgOverlap, checkSupportingDistance), its CSS containment idiom (.object-graphic + object-fit: contain + overflow: visible on .placement), and the 10 gold YAMLs as adversarial fixtures. None of these were promoted to src/ or tests/playwright/.

Direct head-to-head: in 5 of 6 static template vs runtime pairings, the runtime renders closer to a usable scene. The runtime wins on bench, hood, electrophoresis, dense_composition, and instrument; the experiments static template wins on well_plate (where the runtime's six empty_bubble hard fails are exactly the problem). This inverts the user's prior framing: the runtime is the better realised path today, not just the path that got polish.

The recommended posture: keep the runtime mount/dispatch/walker/chrome stack, remove the dead css_native_adapter.ts spike, and promote four contract-compatible items from experiments/ — the .object-graphic containment wrapper, the checkArtworkIntegrity diagnostic, object_footprints.yaml, and the 10 gold YAML stress fixtures. None of these require amending PRIMARY_CONTRACT.md item 3.

7. Runtime shortcomings

Three independent classes of shortcoming surface in the runtime metrics:

7.1 SVG asset gap (48 hard fails)

17.3% of placements still resolve to a single-shape outline or an empty group. The pattern is concentrated: off-state instrument SVGs (power_supply_off, microwave_closed, heat_block_closed, lightbox_off) account for the bulk of generic_outline fails, and well_plate_96 accounts for every empty_bubble fail. These are content/asset gaps, not engine bugs.

7.2 Walker completes only 12/26

Failure modes (FG Appendix):

  • missing DOM affordance for a declared target (8 protocols)
  • waitForSelector timeout on visible-but-timed waits (5 protocols)
  • step_validator passes but next-button affordance does not appear (2 protocols)

None of these are mount failures. All three are walker-instrument or scene-wiring concerns, not layout engine concerns.

7.3 Label overlap (46 pairs)

Concentrates in the SDS-PAGE family (sdspage_extract_gel_from_cassette=4, sdspage_load_sample_single_lane=3, sdspage_prepare_gel_cassette=3, sdspage_run_electrophoresis=3). Production CSS has no label policy; FB Feature 7 identified six worked label-policy CSS variants in experiments/css_native_layout/showcase/label_policies/ that were never applied.

Honest read: the shipping runtime fails its own SVG-completeness gate on 17.3% of placements and only walks through 46% of protocols end-to-end. "Mounts cleanly" is a floor, not a quality claim.

12. What must be scrapped

12.1 css_native_adapter.ts (dead spike in the bundle)

src/scene_runtime/layout/css_native_adapter.ts (FA Concern 1) is imported by adapter.ts but gated by is_css_native_well_plate_zoom_spike_enabled() which defaults false. The file's own header (line 9) says: "Must be removed or replaced before NEW1 promotion." It has been sitting in src/ since 2026-05-21 with no plan-of-record to either de-gate or remove. Every production bundle includes it. Remove it; the bundle gets smaller and the ship-risk of an accidental flag flip goes to zero.

12.2 computeRowSlotSceneLayout (unwired alternate strategy)

FA Concern 3: layout_engine.ts exports computeRowSlotSceneLayout (approx lines 700-799). Re-exported by index.ts as a named export. Not called in adapter.ts, not called in render/scene.ts, not called anywhere in the production path. Either wire it, document its purpose, or delete it. As-shipped it just misleads future readers.

12.3 The "experiments will replace the runtime" framing

FF shows experiments-better in 1 of 6 head-to-heads; the experiments precheck verdict shows 0/10 PASS on its own gold templates and 0/110 on its own stress corpus. The CSS-native render path is not a near-promotion path; it is a research artifact that produced useful diagnostics and CSS idioms. Stop scoping work on the assumption that experiments/css_native_layout/ will be the next render target without a contract amendment (which does not exist and is not approved).

12.4 Showcase concepts and storyboards

FB Feature 7: showcase/concepts/ storyboards (electrophoresis_setup_walkthrough.html, pipette_to_well_storyboard.html, selected_well_highlight.html) demonstrate interaction states that the production dispatch pipeline now handles. hover_reveal_demo.html and selected_well_demo.html show what CSS-alone visual behavior could be, but production already implements these through protocol dispatch. Obsolete; do not promote.

12.5 Direction A and scratch CSS variants

Per experiments/css_native_layout/DECISION_MEMO.md: bench_a.css, bench_b.css, bench_d.css, bench_diorama.css, bench_focusedstage.css, bench_gameboard.css and hood/instrument siblings are gitignored, superseded by Direction B. spike_css_native_well_plate_zoom.mjs inlines a JS mirror of the adapter (divergence risk per file header). Remove the spike test once the spike adapter is removed.

13. What should be kept

13.1 Runtime mount path

26/26 protocols mount cleanly with zero RUNTIME INITIALIZATION ERROR overlays. The mount path through bundle/entry.ts → loader/world.ts → render/scene.ts is the load-bearing achievement of Round 3. Keep.

13.2 Zone-based layout engine

src/scene_runtime/layout/layout_engine.ts (legacyComputeLayout) is the mined-verbatim production engine, historically active since 2026-05-05, with 8 documented functions. It works; FF shows it beats the CSS-native static templates in 5 of 6 pairings. Keep.

13.3 Dispatch (click and adjust)

src/scene_runtime/dispatch/click.ts and src/scene_runtime/dispatch/adjust.ts drove 133 successful interactions. The adjust gesture is the continuous set-point primitive (volume sliders, voltage knobs) that the contract requires must not collapse into click; it is working. Keep.

13.4 Walker engine

tests/playwright/walker/engine.mjs walked 12/26 protocols to completion. Its 14 failures decompose into walker-instrument or scene-wiring concerns, not runtime mount failures. Improve the walker (test-clock affordance, next-button visibility detection); do not replace it.

13.5 Foundation HTML wiring

The per-protocol HTML shells produced by the build pipeline mount the runtime bundle consistently. No foundation-level wiring bugs surfaced in FG. Keep.

13.6 ns0 fix and chrome stack

The chrome stack (src/scene_runtime/chrome/*: prompt, feedback, next button, scene frame, adjust panel) renders without console errors on 24/26 protocols. The two console errors both come from sdspage_run_electrophoresis and are non-blocking. Keep the chrome stack; chase the two errors as a focused fix.

14. What should be promoted from experiments/

Four contract-compatible promotions. None of these require amending PRIMARY_CONTRACT item 3.

14.1 .object-graphic containment wrapper + object-fit: contain

Source: experiments/css_native_layout/styles/bench.css lines 155-163 (.placement { overflow: visible; }), lines 334-358 (.object-graphic centering + object-fit: contain). This is a render-layer CSS rule, not layout engine behavior. PRIMARY_DESIGN.md's no-crop rule currently has no structural enforcement in src/; the rule text is promoted but the CSS mechanism is not. Promote the wrapper plus the three CSS rules into src/style.css and have src/scene_runtime/render/apply.ts emit the wrapper. Cost: tiny. Source: experiments/css_native_layout/precheck.mjs ~lines 1050-1200. The other 11 precheck checks can follow.

14.3 object_footprints.yaml

Source: experiments/css_native_layout/object_footprints.yaml (146 entries). Maps object names to footprint size classes. Required to correctly size objects under any future CSS-native render path. Move to docs/specs/ as authoritative registry or to generated/ as builder output. Does not change runtime behavior today; unblocks future work.

14.4 The 10 gold stress YAMLs

Source: experiments/css_native_layout/stress_scenes/gold/gold_*.yaml. Follow the standard placement/zone vocabulary; register as tests/content/dev_smoke/ fixtures (per PRIMARY_SPEC.md "dev_smoke" protocol_type). Exercise adversarial scene compositions no current smoke fixture covers: dual large equipment competing for instrument_station, repeated identical small items, adversarial size mix. No contract amendment needed.
