# NEW2 no-crop visual audit

Generated: 2026-05-20T21:51:15.807Z

This audit enforces the hard rule that visible SVG assets must never be cropped or aspect-distorted. It runs Playwright against each canonical scene template and measures, per placement, the natural SVG intrinsic size vs. the rendered bounding box and the rendered bounding box vs. the placement card. Three failure types are reported: `off-card` / `cropped` (artwork bbox extends beyond placement bbox), `distorted` (rendered aspect ratio deviates >10% from natural), and `too-small` (rendered area <30% of natural).

Script: `inspect.mjs`

## Root cause notes

1. `.region--work_surface`, `.region--rear_shelf`, `.region--front_tools` set `overflow: visible` (see `bench.css` lines 105-117). The inline comment ('No-crop rule: SVG assets must not be clipped. overflow:visible allows tall glassware to extend beyond the grid row') intentionally lets assets spill out of their card. Under the new hard rule, that is itself a violation: the asset escapes the card it claims, so labels, layout slots, and adjacent objects no longer align with the rendered art.

2. `.object-graphic img` uses `width: auto; height: auto; max-width: 100%; max-height: 100%; object-fit: contain` (bench.css lines 331-340). When the placement allows visible overflow, `max-height: 100%` is computed against a parent with no enforced height clip, so the image renders at its full natural-aspect-scaled size and extends top + bottom of the card.

3. Several footprints (e.g., `.footprint--small-tool` = 50x60..80x90, `.footprint--container` = 220x240..320x360) have widths and heights that do not match common asset aspect ratios (p200 micropipette is 34x150, ratio 0.23; well_plate_96 is 393x279, ratio 1.41). This produces `too-small` artifacts when the card is far wider than the asset needs (tip_box, kimwipe_pad, counter_slide_cartridge) and `distorted` artifacts when card width/height ratio fights the asset's viewBox.

## Recommended next fix order

1. **Fix bottle footprint** (`.footprint--handheld`, 90x110..130x160) -- bottles are the single largest failure family (rear*shelf rows in `crowded_bench_dense`, `drug_dilution*\*`, `electrophoresis_bench`, `staining_bench`). All bottle assets are natural 54x150 to 69x150; current 90-wide x 160-tall card forces the SVG to render at ~230 tall with `object-fit:contain`math, spilling top and bottom. Set`.footprint--handheld { min-height: 230px; max-height: 260px; }`or reduce`max-width` so contain-scaling caps height at the card.

2. **Remove `overflow: visible` overrides** in `.region--work_surface` etc. The 'No-crop rule' comment is now wrong under the new hard rule. The correct enforcement is: card sized to fit asset, not card lying about its bounds.

3. **Fix electrophoresis_bench** (9 incidents, all large equipment). `.footprint--instrument` and `.footprint--large-equipment` need height bumps to host the tank + gel cassette + electrode module without bottom overflow.

4. **Fix `crowded_bench_dense`** (8 incidents, 7 bottles + kimwipe). Same root cause as #1; density override `.scene-container[data-scene-density=crowded] .scene--bench .footprint--handheld` (63x77..91x112) is even tighter and amplifies the spill.

5. **Tip-box / cartridge tiny-render fix**: `tool_tips` (drug_dilution_plate_workspace) renders at 3.3% of natural area; `tool_tip_box` (drug_dilution_workspace_dense) at 8.7%; `front_tools_counter_slide_cartridge` at 8.6%. These cards are sized as `.footprint--small-tool` (50x60..80x90) but the assets are wide (212x150, 230x150). Reclassify these placements to a wider footprint or add `.footprint--wide-small-tool` with min-width:140px.
