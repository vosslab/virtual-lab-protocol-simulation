## Policy proposal

### Which blob types should be tracked vs gitignored

1. PNGs that are ESSENTIAL_EVIDENCE (specifically BEFORE/AFTER comparison pairs documenting a specific code-state transition) - strongest candidates for tracking.
2. PNGs that are REGENERABLE_SCREENSHOT at current code state - NOT tracked. Gitignored snapshot folder or generated on demand.
3. PDFs reproducible from tracked HTML via tools/html_to_pdf.mjs - NOT tracked.
4. HTML gallery wrappers generated by pipeline tooling - NOT tracked. Add to .gitignore.

### Which evidence categories warrant tracking

Track:

- BEFORE/AFTER pairs where BEFORE requires now-reverted code state
- Concept storyboards / hand-authored diagrams with no automated regen path
- Annotated screenshots with manual annotations

Do not track:

- Full-scene gallery renders (stable pipeline outputs)
- Per-object isolation crops from current code state
- PDF snapshots with live source HTML
- HTML gallery wrappers from tooling

### When to use external storage

- Evidence bundles >1 MB per plan: gitignored ZIP snapshots in docs/evidence_bundles/ (gitignored), referenced by filename in plan MD.
- Formal release-quality evidence archive: separate evidence branch or attached GitHub release artifact, not tracked files on main.
- LFS not warranted at current scale.

### How to reference visual evidence without bloating history

Plan MD files reference screenshots by relative path + comment block when asset is regenerable:

```markdown
<!-- REGENERABLE: run tests/playwright/spike_css_native_well_plate_zoom.mjs -->

!`new0_well_plate_96_zoom.png`
```

For gitignored evidence: \_EVIDENCE_MANIFEST.md stub in plan directory noting what snapshots exist locally, how to regen, what code state.

### How to keep audit trails when evidence is not tracked

Plan MD snapshot metadata block:

```markdown
## Evidence snapshot

- Taken at: git commit <short-hash>
- Script: tests/playwright/spike\_\*.mjs
- Regenerate: node tests/playwright/<script>.mjs
- Stored locally (gitignored): docs/active_plans/\_screenshots/<plan_prefix>/
```

Preserves audit trail in tracked text while keeping binaries out of history.
