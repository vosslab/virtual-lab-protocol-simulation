## Key subtrees

### `content` - Authored YAML

Protocols, scenes, and objects. All authored YAML; nothing under `content/` is generated.
The canonical layout rule lives in [TARGET_FILE_STRUCTURE.md](specs/TARGET_FILE_STRUCTURE.md#protocol-cluster-layout).

| Folder        | Purpose                                                                                                   |
| ------------- | --------------------------------------------------------------------------------------------------------- |
| `protocols`   | Mini-protocols and sequence runners, organized by topic cluster (`cell_culture/`, `sdspage/`, `runners/`) |
| `base_scenes` | Reusable base scenes shared across protocols                                                              |
| `objects`     | Reusable lab object definitions, grouped by kind (bottle, equipment, decoration, etc.)                    |

### `pipeline` - Protocol and scene builder

Regenerates gitignored `generated/` artifacts from authored YAML and SVG sources.

| File                                                           | Purpose                                                                                              |
| -------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| [bootstrap_generated.sh](../pipeline/bootstrap_generated.sh)   | Single entry point: regenerates all protocol, scene, inventory, and SVG generated artifacts          |
| [build_protocol_data.py](../pipeline/build_protocol_data.py)   | YAML -> generated TS; eight schema rules; `--protocol`, `--validate-only`                            |
| [build_scene_data.py](../pipeline/build_scene_data.py)         | Compile scene YAML to generated/scene_data.ts                                                        |
| [generate_svg_globals.py](../pipeline/generate_svg_globals.py) | Regenerate `generated/svg_assets/*.ts` and `generated/svg_manifest.ts` from `assets/equipment/*.svg` |

### `validation` - Protocol and scene validators

Standalone YAML validators and audit tools for schema, structure, and logical consistency.

#### `yaml` - Protocol and scene YAML validator

| File                                                                          | Purpose                                                                         |

| [protocol_validator.py](../validation/yaml/protocol_validator.py)             | Protocol schema and interaction validation                                      |
| [object_validator.py](../validation/yaml/object_validator.py)                 | Object schema and state_field validation                                        |
| [scene_base_validator.py](../validation/yaml/scene_base_validator.py)         | Scene YAML base schema validation                                               |
| [scene_protocol_validator.py](../validation/yaml/scene_protocol_validator.py) | Scene-protocol adapter rules (target resolution)                                |
| [protocol_audit.py](../validation/yaml/protocol_audit.py)                     | Audit unused materials, items, and cross-protocol orphans                       |

#### `stepper` - Protocol step-flow validator

| File                                                 | Purpose                                                           |
| ---------------------------------------------------- | ----------------------------------------------------------------- |
| [step_check.py](../validation/stepper/step_check.py) | Canonical stepper entry point; walker that executes protocol flow |
| [runner.py](../validation/stepper/runner.py)         | Step-by-step protocol walker and state tracker                    |
| [flow.py](../validation/stepper/flow.py)             | Protocol flow resolution (entry_step, next_step chaining)         |

#### `svg` - SVG asset validator

#### `manual` - Human-readable protocol manual

| File                                                                                   | Purpose                                 |
| -------------------------------------------------------------------------------------- | --------------------------------------- |

Focused Playwright browser checks live one level up in
`playwright`, including layout, scoring,

full-path walker and its helpers.

#### `e2e` - Non-browser E2E (reserved)

Contains standalone non-browser E2E runners such as
[e2e_bandit_security.py](../tests/e2e/e2e_bandit_security.py) and
[e2e_facade_smoke.py](../tests/e2e/e2e_facade_smoke.py). New shell or
Python E2E runners belong here as `e2e_*.sh` or `e2e_*.py`. The legacy
single-file parity snapshot lives under
`tests/fixtures/`.

### `assets` - Source art

`assets/equipment/` contains the tracked source SVG art consumed by
[generate_svg_globals.py](../pipeline/generate_svg_globals.py). Sidecar
`*.colormap.json` files name semantic groups of ids for the recolor pipeline.
The `generated/` tree is git-ignored; it is regenerated from these source
SVGs by build and test scripts.

### `servier` - Pristine Servier source SVGs

Reference art used as input for derived equipment SVGs. Do not edit
in place.

### `devel` - Maintainer scripts

| File                                                | Purpose                                  |
| --------------------------------------------------- | ---------------------------------------- |
| [setup_playwright.sh](../devel/setup_playwright.sh) | Idempotent Playwright (chromium) install |
| [commit_changelog.py](../devel/commit_changelog.py) | Helper for changelog hygiene             |

### `experiments` - Transient prototypes

Non-production prototypes outside the layout-engine path. Transient: current contents are subject to removal once integration spikes resolve.

### Other repos

`OTHER_REPOS` holds sibling-repo checkouts for
cross-reference. Subdirectories are git-ignored.

## Where to add new work

| Type                  | Location                                                                                          |
| --------------------- | ------------------------------------------------------------------------------------------------- |
| Game runtime          | New `.ts` file under `src`, imported from `init.ts` or a scene                                    |
| New scene             | `scenes` with scene YAML, an adapter, a side-effect import in `init.ts`, and a render switch case |
| New step UI / emitter | `steps`                                                                                           |
| Build/audit tooling   | `tools` (Python, single-purpose)                                                                  |
| Game styles           | [style.css](../src/style.css)                                                                     |
| Pytest tests          | `tests/test_*.py`                                                                                 |
| Browser tests         | `tests/playwright/test_*.mjs` (smoke) or `tests/playwright/e2e/*.mjs` (walkthroughs)              |
| Non-browser E2E       | `tests/e2e/e2e_*.sh` or `e2e_*.py`                                                                |
| Documentation         | `docs/` with SCREAMING_SNAKE_CASE filename                                                        |
