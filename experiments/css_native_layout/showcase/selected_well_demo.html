How this ties to the runtime

In production, each well is a scene-object subpart whose declared state fields (material_name, material_volume) drive its visual state via the object's visual_states map; selection is a transient runtime flag on the addressed target.

A validated click interaction emits a response whose scene_operations include an ObjectStateChange that writes the new material field on the well subpart.

The renderer then re-resolves that subpart's visual_states entry and swaps fill/stroke without any per-protocol scene code — this static mockup shows the three terminal looks the renderer must produce.
