CONCEPT DEMO -- NOT PRODUCTION, NOT WIRED TO RUNTIME

Concept: Electrophoresis Setup Walkthrough

Six-stage setup of a gel box: pour gel, insert comb, load buffer, load samples, attach leads, set voltage.

STAGE 1
Pour molten gel
Validator target_with_value confirms gel weight matches recipe. ObjectStateChange sets material_volume on tray.
STAGE 2
Insert well comb
Comb is a scene object placed via the layout engine. Removing it later triggers ObjectStateChange exposing the well lanes.
STAGE 3
Load running buffer
Buffer level passes target_with_value threshold (covers gel by 2 mm). Visual fill is an object visual_state, not a separate primitive.
STAGE 4
Load samples into wells
Each well lane is a discrete target (e.g., gel.lane_5). Loading dispatches ObjectStateChange setting that lane's material_name.
STAGE 5
Attach power leads
Validator correct_target on each socket. Polarity errors trigger an error highlight state (see selected-well concept).
STAGE 6
Set 120 V, start run
adjust gesture on voltage knob. TimedWait primitive runs the gel; emits gel_box_elapsed event when complete.
What this concept shows: a six-stage storyboard for the electrophoresis setup that maps each visible bench action onto an existing protocol primitive (ObjectStateChange, TimedWait, validator presets). It demonstrates how a multi-station workflow can be authored as a sequence of single-target interactions without inventing new vocabulary -- gel lanes are addressed individually, leads use correct_target with polarity error feedback, and the voltage knob is an adjust gesture. To realize this in production, the gel-box scene needs subpart lanes as addressable targets, a power-supply object with a numeric set_voltage field, an error highlight overlay for wrong polarity, and a TimedWait-driven band migration animation tied to the gel-box elapsed event. The storyboard is intentionally geometry-rough; the production layout would come from the layout engine, not from the hand-coded SVG shown here.
